# Notion_Fishing

Diese Ressource wurde als interaktives Fischereiskript für ESX/QBCore-Server erstellt..

## Features
- Sofort kompatibel mit ESX und QBCore
- Optimierte Nutzung von 0,00 ms im Leerlauf
- Erfolg durch Überprüfung der Fähigkeiten
- Vollständige Animationen und Requisiten
- Wahrscheinlichkeit, dass die Angelrute bei fehlgeschlagenem Fertigkeitscheck bricht (kann in der Konfiguration geändert werden)
- Konfigurierbare zufällige Wartezeit für den Biss auf der Leitung
- Konfigurierbare Angelbelohnungen (standardmäßig 4 enthalten)
- Konfigurierbare Preise für den Verkauf von Angelbelohnungen
- Konfigurierbarer Schwierigkeitsgrad der Fertigkeitsprüfung pro Angelbelohnung
- Möglichkeit, vom Boot, Pier oder jedem anderen Ort mit Gewässer aus zu angeln
- Keine beruflichen Anforderungen
- Vollständig konfigurierbarer Fischkäufer zum Verkauf von Fisch

## Dependencies
- es_extended ODER qb-core
- ox_lib - https://github.com/overextended/ox_lib/releases


## Installation

- Stellen Sie sicher, dass Sie über Abhängigkeiten verfügen.

- Stellen Sie sicher, dass die Elemente zum Server hinzugefügt werden (überprüfen Sie das Verzeichnis „Inventory_Items“ auf Bilder/Beispiele).

- Skript in Ihr Verzeichnis „resources“ kopieren

- Fügen Sie „ensure notion_fishing“ in Ihre „server.cfg“ ein (*nach* den Abhängigkeiten).

### Extra Information
- Stellen Sie sicher, dass `ox_lib` vor `notion_fishing` gestartet wird.
- Im Verzeichnis „Inventory_Items/InventoryImages“ enthaltene Inventarbilder
- Sie müssen den Artikel hinzufügen. `fishingrod` und `fishbait` in einem Ihrer Shops im Spiel oder an einem Ort, an dem Ihre Spieler es erhalten können.
